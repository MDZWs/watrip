# 社区参考旅程预览 & 出行方式提示 - Product Requirement Document

## Overview
- **Summary**: 增强社区参考旅程的浏览体验，支持行程预览（列表+地图双模式）、一键套用后编辑，以及在行程的两个景点之间增加出行方式选择和距离/时间提示，通过高德地图API计算步行、骑行、驾车、公交等多种出行方式的距离和耗时。
- **Purpose**: 让用户在社区浏览参考行程时能先预览行程详情（含地图路线）再决定是否套用，提升参考行程的可用性；同时在行程规划中为用户提供景点间的出行建议，帮助用户合理安排时间和交通。
- **Target Users**: 使用圆周·AI旅行规划应用的旅行者，尤其是希望参考他人行程、关注出行便利性的用户。

## Goals
- 用户点击社区行程卡片时，先进入预览页而非直接套用
- 预览页支持列表模式和地图模式切换
- 用户可在预览页选择"套用行程"后进入可编辑状态
- 为种子模板行程补充完整的dayPlans景点数据
- 在行程的相邻景点之间显示出行方式提示卡片（距离、时间、交通方式选择）
- 通过高德地图API支持步行、骑行、驾车三种出行方式的路径和耗时计算
- 用户可在出行提示卡片上切换出行方式，时间自动更新

## Non-Goals (Out of Scope)
- 用户发布自己的行程到社区（社区发布功能）
- 实时公交/地铁班次查询
- 打车费用估算
- 多路线对比展示（每次仅展示当前选中方式的最优路线）
- 社区行程的评论/点赞交互优化

## Background & Context
- 当前项目已有社区页面（communityPage），展示瀑布流卡片，但点击卡片直接调用App.useTemplate()将目的地填入输入页，用户无法预览行程详情
- 当前模板数据（seed templates）只有元信息（标题、作者、标签等），没有完整的dayPlans景点数据
- 地图模块（MapModule）已支持驾车（AMap.Driving）和步行（AMap.Walking）路径规划，但缺少骑行（AMap.Riding）支持
- 行程详情页的时间轴中，相邻景点之间通过轴线连接，但没有显示交通方式和距离信息
- 高德JS API v2.0已在index.html中加载，key已配置

## Functional Requirements

### FR-1: 社区行程预览页
- **FR-1.1**: 点击社区瀑布流中的行程卡片，进入行程预览页，而非直接套用
- **FR-1.2**: 预览页顶部显示行程基本信息：封面、标题、作者、天数、目的地、标签、简介、被复制次数、点赞数
- **FR-1.3**: 预览页底部有两个操作按钮："套用此行程"（主按钮）和"返回"（次按钮）
- **FR-1.4**: 预览页支持列表模式和地图模式切换，使用浮动按钮切换（类似现有trip detail的模式切换）
- **FR-1.5**: 列表模式展示完整的每日行程时间轴，包含景点、时间安排、介绍
- **FR-1.6**: 地图模式展示景点标记和路线连线，可切换天数查看
- **FR-1.7**: 预览页中的行程数据为只读展示，不可编辑、不可拖拽、不可调整时间
- **FR-1.8**: 点击"套用此行程"后，将该行程复制为用户自己的行程，跳转至行程详情页并进入可编辑状态

### FR-2: 种子模板行程数据增强
- **FR-2.1**: 为现有6个种子模板补充完整的dayPlans数据，包含每日景点（含名称、地址、emoji、建议游玩时长、时间段安排）
- **FR-2.2**: 每个模板的dayPlans需包含合理的景点顺序和时间安排
- **FR-2.3**: 模板数据包含城市信息，用于地图地理编码

### FR-3: 景点间出行方式提示
- **FR-3.1**: 在行程详情页（编辑模式）的时间轴中，相邻两个景点之间显示出行提示卡片
- **FR-3.2**: 出行提示卡片显示：距离（如"2.3km"）、预估时间（如"8分钟"）、当前出行方式图标
- **FR-3.3**: 出行提示卡片提供出行方式切换选项：步行、骑行、驾车、公交
- **FR-3.4**: 用户切换出行方式时，通过高德API重新计算距离和时间，更新显示
- **FR-3.5**: 骑行方式使用AMap.Riding插件计算路径和时间
- **FR-3.6**: 如果API调用失败，使用直线距离和平均速度估算作为降级方案
- **FR-3.7**: 预览模式下的行程也应显示出行提示，但不可编辑切换（仅展示推荐方式）
- **FR-3.8**: 出行时间应考虑到前后景点的时间安排，作为用户调整时间的参考

### FR-4: 高德骑行路径规划集成
- **FR-4.1**: 在高德地图SDK加载中增加AMap.Riding插件
- **FR-4.2**: MapModule中增加Riding服务实例
- **FR-4.3**: 新增通用路径计算方法，支持driving/walking/riding/transit四种方式
- **FR-4.4**: 路径计算结果需包含距离（米）和耗时（秒）

## Non-Functional Requirements
- **NFR-1**: 出行方式切换响应时间应在1秒内（高德API正常响应情况下）
- **NFR-2**: 预览页加载时间不应超过2秒（地理编码可渐进加载）
- **NFR-3**: 地理编码结果应有缓存，避免重复请求相同地点
- **NFR-4**: 界面设计应与现有设计风格保持一致，使用相同的配色和组件风格
- **NFR-5**: 出行提示卡片设计应紧凑美观，不喧宾夺主

## Constraints
- **Technical**: 纯前端实现（HTML/CSS/JS），使用高德地图JS API v2.0，无后端服务
- **Business**: 使用已有的高德API Key（f517b3497f7c40a755d595fd8038156b）
- **Dependencies**: 高德地图JS API（已集成）、localStorage存储

## Assumptions
- 高德AMap.Riding插件在v2.0版本中可用且接口与Driving/Walking类似
- 公交换乘（AMap.Transfer）查询较复杂，初版可先实现步行、骑行、驾车三种，公交作为后续扩展或使用估算
- 种子模板数据使用硬编码方式内置，不需要从服务端获取
- 套用行程时，需要对模板中的景点进行高德地理编码以获取坐标，此过程可异步进行

## Acceptance Criteria

### AC-1: 社区卡片点击进入预览页
- **Given**: 用户在社区发现页
- **When**: 点击任意一个社区行程卡片
- **Then**: 进入该行程的预览页，显示行程封面、标题、作者、天数、标签、简介等基本信息，底部有"套用此行程"按钮
- **Verification**: `human-judgment`

### AC-2: 预览页列表模式展示完整行程
- **Given**: 用户在预览页列表模式
- **When**: 页面加载完成
- **Then**: 显示每日行程的时间轴，包含景点名称、emoji、时间、简介，内容只读不可编辑
- **Verification**: `human-judgment`

### AC-3: 预览页地图模式切换
- **Given**: 用户在预览页
- **When**: 点击"地图模式"浮动按钮
- **Then**: 切换到地图视图，显示景点标记和路线连线，地图上可切换天数查看
- **Verification**: `human-judgment`

### AC-4: 套用行程功能
- **Given**: 用户在预览页
- **When**: 点击"套用此行程"按钮
- **Then**: 行程被复制到用户的行程列表，跳转至行程详情页，行程可编辑（添加/删除/调整景点、修改时间等）
- **Verification**: `programmatic`

### AC-5: 景点间出行提示卡片显示
- **Given**: 用户在行程详情页（或预览页），当天行程有2个及以上景点
- **When**: 行程渲染完成
- **Then**: 相邻景点之间显示出行提示卡片，包含距离、预估时间、出行方式图标
- **Verification**: `human-judgment`

### AC-6: 出行方式切换
- **Given**: 用户在行程详情页，出行提示卡片可见
- **When**: 点击出行提示卡片上的不同出行方式（步行/骑行/驾车）
- **Then**: 通过高德API重新计算该方式的路径，卡片上的距离和时间更新为对应方式的数据
- **Verification**: `programmatic`

### AC-7: 骑行路径计算
- **Given**: MapModule已初始化
- **When**: 调用骑行路径规划方法计算两点之间
- **Then**: 返回骑行的距离（米）和时间（秒），与高德Riding服务返回一致
- **Verification**: `programmatic`

### AC-8: 种子模板包含完整行程数据
- **Given**: 系统加载社区模板
- **When**: 读取种子模板数据
- **Then**: 每个模板包含完整的dayPlans数组，每天有theme、spots数组，每个spot有name、emoji、duration、startMin/endMin等字段
- **Verification**: `programmatic`
