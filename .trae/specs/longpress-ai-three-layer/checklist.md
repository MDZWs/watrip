# 长按优化与AI推荐三层架构重构 - Verification Checklist

## 长按删除优化
- [ ] 手指在卡片上移动10px以内时长按不会取消（容差生效）
- [ ] 手指移动超过10px时长按正确取消
- [ ] 桌面端鼠标移动也遵循同样的容差逻辑
- [ ] 长按时有明确视觉反馈（缩放+半透明效果）
- [ ] 长按时设备震动（支持vibrate的设备）
- [ ] 松开后弹出自定义样式确认弹窗（不是原生confirm）
- [ ] 弹窗显示行程名称
- [ ] 点击"取消"按钮关闭弹窗，行程不删除
- [ ] 点击遮罩区域关闭弹窗，行程不删除
- [ ] 点击"删除"按钮删除行程并刷新列表，显示toast
- [ ] 删除后列表正确更新，卡片消失
- [ ] 正常短按卡片仍然跳转到行程详情页
- [ ] 长按后松开不会触发卡片点击跳转（无点透问题）

## AI三层推荐架构
- [ ] MapModule.searchNearbyPOIs方法存在并能正常调用
- [ ] searchNearbyPOIs返回的每个POI距离都≤20km（硬约束生效）
- [ ] searchNearbyPOIs返回结果数量≤5（Top5）
- [ ] 美食POI按评分+距离综合排序，景点按距离升序
- [ ] 同名POI去重，只保留最近的一个
- [ ] _getReferencePoint优先取当天最后一个有坐标的景点
- [ ] 当天无有坐标景点时回退到城市中心
- [ ] 城市中心通过MapModule.getCityCenter获取
- [ ] 点击"找美食"按钮触发三层流程（先搜高德POI，再给AI润色）
- [ ] 点击"找酒店"按钮触发三层流程
- [ ] 点击"找体验"按钮触发三层流程
- [ ] AI收到的prompt包含明确约束：只能使用提供的POI，不得新增/改名
- [ ] 最终展示的推荐卡片坐标来自高德真实数据（不使用AI返回的坐标）
- [ ] 推荐卡片显示距离标签（如"📍 0.3km"）
- [ ] 点击推荐卡片的"+ 添加"按钮能正常添加到行程，且有正确坐标
- [ ] AI返回的name不在候选列表中时被过滤丢弃
- [ ] 搜索无结果时显示"附近20km内未找到合适的{类型}..."提示
- [ ] 网络异常或API失败时不崩溃，显示友好提示
- [ ] 三层流程AI温度参数为0.3（更确定性）
- [ ] 路线优化(optimize)、避坑提示(tips)等非附近推荐功能保持原有逻辑不变

## 通用
- [ ] 页面加载无JavaScript错误
- [ ] 控制台无报错
- [ ] 附近推荐响应速度可接受（2秒内显示结果）
- [ ] 已有行程数据不受影响（兼容旧数据）
