# 旅行 App 社区化升级 - 产品概述

> 文档版本：v1.0
> 生成时间：2026-07-02
> 阶段：需求澄清完成 / 待技术选型
> 工作流：SpecForge 项目级 - Skill 2

---

## 1. 项目愿景

把现有旅行规划 App 从"个人规划工具"升级为"创作者社区 + 内容平台"。让每个用户都能设计自己的旅行专栏，通过创作价值获得关注、积分与成长，并通过吐槽打卡系统形成地点维度的真实内容沉淀，最终构建一个"规划—创作—分享—参考—再规划"的旅行内容闭环。

---

## 2. 核心价值

1. **人人可创作的旅行专栏**：用户可发布两类内容——行程模板（结构化行程，可一键套用）和图文游记（图文+可挂载行程+可标注地点），所有内容聚合到个人主页。
2. **创作者价值体系**：通过"关注/参考/采纳/点赞/收藏"五种行为衡量创作价值，转化为积分与经验，驱动等级成长与功能特权解锁。体系包含三套互不挂钩的机制（成长引导、道具彩蛋、投票投流）。
3. **吐槽打卡系统**：用户在地点定位打卡并可选写吐槽，形成"专题区域"，所有打卡公开可见，配合合规审核机制保障内容健康。

---

## 3. 关键架构决策

### 3.1 后端方案：引入 BaaS
- 项目从纯前端 + localStorage 升级为 **BaaS（后端即服务）** 架构
- 需要支撑：用户认证、内容存储与分发、关注关系、积分账本、打卡内容、审核队列
- 具体选型待技术栈阶段确定（候选：Supabase / Firebase）

### 3.2 保留现有机制
- 纯前端 HTML/CSS/JS 主体保留
- 高德地图 API 已集成（POI、地理编码、路径规划）
- 行程规划核心（列表/地图双模式、时间轴、AI 助手）保持不变
- AI 始终定位为助手，**不限制规划次数**，不作为积分特权

## 5. 系统核心板块

### 板块一：旅行专栏

#### 5.1.1 内容类型
两类并列内容，用户发布时自选：

**A. 行程模板**
- 数据结构沿用现有社区模板（dayPlans、景点、时间轴、tags、season、suitableFor 等）
- 发布流程：用户在行程详情页点"发布到我的专栏"即可一键发布
- 读者可预览（列表/地图双模式）后一键套用

**B. 图文游记（C 形态：图文 + 可挂载行程 + 可标注地点）**
- 封面图 + 标题 + 正文（富文本，可插图）+ 标签
- 可选挂载一份自己的行程模板（读者可跳转预览）
- 正文中可标注地点（带坐标，读者可点击查看地图位置）
- 图片来源（分阶段）：
  - 第一阶段：从高德 POI 图片库选图（零成本快速填充）
  - 第二阶段：增加用户上传（真实感）

#### 5.1.2 个人主页（形态三）
- 每个用户有专属主页，聚合展示：发布的所有行程模板 + 图文游记 + 打卡记录
- 显示：总关注数、总点赞数、总被套用数、等级徽章、道具收藏柜
- 别人可从此页"关注"创作者

#### 5.1.3 内容审核
- 图文游记采用"先发后审"——发布即可见，自动审核（关键词/图片识别）+ 用户举报触发人工复审

### 板块二：创作者价值体系（三套互不挂钩的机制）

#### 5.2.1 机制一：成长引导（官方任务清单 → 经验 → 功能特权）

**核心逻辑**：积分 → 灌入经验池（1 积分 = 1 经验）→ 经验达标升级 → 等级解锁功能特权。

**五种行为的价值权重**（创作者获得积分）：

| 行为 | 积分 | 说明 |
|------|------|------|
| 被参考（查看） | +1 | UV 去重，同用户同天只算一次 |
| 被点赞 | +3 | 同用户对同内容只计一次 |
| 被收藏 | +5 | 同用户对同内容只计一次 |
| 被采纳（套用） | +10 | 他人套用行程模板 |
| 被关注 | +15 | 他人关注创作者 |

**新手任务清单**（完成约 350 经验，解锁 Lv4 PDF 导出）：

| 任务 | 经验 |
|------|------|
| 完善个人资料 | 20 |
| 创建第一个行程 | 40 |
| 添加 3 个景点 | 30 |
| 调整一次景点时间 | 30 |
| 切换到地图模式 | 20 |
| 查看景点间出行方式 | 30 |
| 使用行李清单 | 20 |
| 使用记账 | 20 |
| 套用一个社区模板 | 30 |
| 发布第一篇图文游记 | 60 |
| 点赞一个作品 | 10 |
| 收藏一个作品 | 20 |
| 关注一个创作者 | 20 |

**功能特权清单**（按等级解锁，非道具）：

| 等级 | 解锁特权 |

#### 5.2.2 机制二：道具彩蛋系统（不挂钩功能，纯收集）

- 道具**不与等级/权限/经验挂钩**
- 作为彩蛋形式给到用户，纯收集乐趣 + 身份标识
- **每月旅行主题活动**：出专属限定道具（如夏日海岛季的"贝壳"、秋日 CityWalk 季的"落叶"）
- 完成活动任务获得限定道具
- 道具展示在个人主页"收藏柜"
- 集齐 N 个限定道具 → C 阶段可兑换实物周边（闭环预告）
- 偶有随机彩蛋掉落（登录、节日），增加惊喜感

#### 5.2.3 机制三：投票投流（签到 → 票 → 给作品投票）

- "票"是独立货币，与积分**不互通**
- 每日签到获得"票"
- 用户用票给好作品投票
- 票数高的作品获得更多曝光
- 投票人（签到活跃）与被投票人（作品获票→曝光→被看→赚积分）形成正循环

### 板块三：吐槽打卡系统

#### 5.3.1 打卡与吐槽的关系（C 形态）
- 打卡是行为，吐槽是可选结果
- 用户在地点定位打卡 → 打卡后可选"写吐槽"
- 打卡记录"谁在什么时候来过"
- 写了吐槽的打卡突出展示在专题区域
- 没写吐槽的打卡只显示在头像墙（"X 人来过"）

#### 5.3.2 地址来源（C 形态：两者都支持，区分标识）
- **实地打卡**：GPS 定位，匹配最近 POI，需在地点 500 米内，带"实地"标识（可信度高）
- **云打卡**：手动搜索/选择地点，不要求在现场，带"云打卡"标识（可信度低，不进实地排行榜）

#### 5.3.3 专题区域形态（C 形态：地图聚合 + 话题页结合）
- 入口：地图上点地点进入，或社区话题列表进入
- 页面结构：顶部地点信息（地图小窗 + 地址 + 打卡人数）+ 下方吐槽瀑布流
- 每个地点有专属聚合页，类似"#杭州西湖#"

#### 5.3.4 合规机制
- **审核方式**：先发后审——发布即可见，自动审核（关键词/图片识别）+ 用户举报触发人工复审
- **违规分级**：

| 级别 | 示例 | 处理 |
|------|------|------|
| 轻度 | 广告引流、水文、跑题 | 内容降权，警告通知 |
| 中度 | 不实信息、轻微人身攻击、低俗 | 内容隐藏，警告 + 记违规 |
| 重度 | 违法、暴力、色情、政治敏感 | 内容删除，封禁 7 天 |
| 极重 | 多次重度违规 | 永久封禁，积分清零 |

- **举报流程**：用户举报 → 进复审队列 → 人工判定 → 通知双方；同一内容被 3 人举报自动临时隐藏；恶意举报反向惩罚
- **违规零容忍**：吐槽被举报违规 **-30 分**（重惩罚）

#### 5.3.5 打卡与积分联动

| 行为 | 积分 |
|------|------|
| 发布打卡（实地） | +3 |
| 发布打卡（云打卡） | +1 |
| 写吐槽（带图文） | +5 |
| 吐槽被点赞 | +2/次 |
| 吐槽被收藏 | +3/次 |
| 吐槽被举报违规 | -30/次 |

**防刷**：同一地点 24 小时内只能打卡一次；每日打卡积分上限 5 个地点封顶。

## 6. 验收标准（AC）

### AC-专栏-1：发布行程模板到专栏
- **Given**：用户在行程详情页，已完成一个行程
- **When**：点击"发布到我的专栏"
- **Then**：行程作为模板发布到个人主页，出现在社区瀑布流，获得 likes/copies/favorites 计数
- **Verification**：human-judgment

### AC-专栏-2：发布图文游记
- **Given**：用户在创作入口
- **When**：选择"发布图文游记"，填写封面+标题+正文，可选挂载行程模板、标注地点
- **Then**：游记发布到个人主页和社区，读者可查看并跳转挂载的行程
- **Verification**：human-judgment

### AC-专栏-3：个人主页聚合展示
- **Given**：任意用户访问某创作者主页
- **When**：页面加载
- **Then**：展示其所有行程模板、图文游记、打卡记录，及关注数/点赞数/被套用数/等级/道具柜
- **Verification**：human-judgment

### AC-积分-1：积分正确累计
- **Given**：创作者发布内容
- **When**：他人对内容执行查看/点赞/收藏/套用/关注
- **Then**：创作者积分按权重表正确累计，防刷机制生效
- **Verification**：programmatic

### AC-积分-2：等级升级解锁特权
- **Given**：用户经验达到某等级阈值
- **When**：经验灌入达标
- **Then**：自动升级，解锁对应功能特权（如 Lv4 解锁 PDF 导出）
- **Verification**：programmatic

### AC-积分-3：新手任务清单引导
- **Given**：新用户首次进入
- **When**：完成新手任务清单全部任务
- **Then**：经验约达 350，刚好解锁 Lv4 PDF 导出功能
- **Verification**：programmatic

### AC-投票-1：签到获得票并投票
- **Given**：用户每日签到
- **When**：获得票后对某作品投票
- **Then**：票数扣减，作品票数增加，高票作品获得曝光加权
- **Verification**：programmatic

### AC-打卡-1：实地定位打卡
- **Given**：用户在某地点 500 米内
- **When**：发起实地打卡
- **Then**：GPS 验证通过，打卡成功，带"实地"标识，计入实地排行榜
- **Verification**：programmatic

### AC-打卡-2：写吐槽并展示在专题区域
- **Given**：用户已打卡某地点
- **When**：选择写吐槽（图文）并发布
- **Then**：吐槽出现在该地点专题区域瀑布流，他人可见、可点赞收藏
- **Verification**：human-judgment

### AC-打卡-3：专题区域聚合页
- **Given**：任意用户访问某地点专题区域
- **When**：页面加载
- **Then**：顶部显示地点信息（地图小窗+地址+打卡人数），下方显示吐槽瀑布流与打卡头像墙
- **Verification**：human-judgment

### AC-合规-1：违规内容处理
- **Given**：吐槽内容命中高风险关键词或被 3 人举报
- **When**：触发审核
- **Then**：内容自动隐藏进人工复审，判定违规后按分级处理，作者扣 30 分
- **Verification**：programmatic

### AC-道具-1：活动道具获取
- **Given**：每月主题活动上线
- **When**：用户完成活动任务
- **Then**：获得限定道具，展示在个人主页收藏柜
- **Verification**：programmatic
