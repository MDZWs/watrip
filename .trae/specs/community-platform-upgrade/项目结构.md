## 1. 设计原则

1. **零迁移**：现有 `css/` 和 `js/` 保持不变，不重构现有代码
2. **特性架构（Feature-based）**：新功能按业务模块组织，每个模块是独立单元
3. **第三方隔离**：外部依赖（Supabase 等）统一封装在 `lib/`，便于替换
4. **公共抽取**：跨模块复用的工具放 `shared/`，避免重复代码
5. **渐进引入**：新模块通过 `index.html` 按需引入，不影响现有页面加载

## 3. 模块职责说明

### 3.1 lib/ — 第三方集成封装
| 文件 | 职责 |
|------|------|
| supabase-client.js | Supabase 客户端单例初始化，导出 `supabase` 实例供各模块使用 |
| amap-helper.js | 高德 API 封装：POI 搜索、地理编码、逆地理编码、路径规划、POI 图片获取 |

### 3.2 shared/ — 公共工具
| 文件 | 职责 |
|------|------|
| auth.js | 登录/登出/注册/当前用户状态/登录态持久化/路由守卫 |
| storage-migrate.js | 一次性迁移脚本：把 localStorage 里的行程数据导入 Supabase |
| points-core.js | 积分增减的原子操作、UV 去重校验、防刷校验、经验灌入 |
| ui-kit.js | toast 提示、确认弹窗、加载态、空状态组件 |

### 3.3 modules/community/ — 旅行专栏
| 文件 | 职责 |
|------|------|
| community-page.js | 社区瀑布流主页，展示行程模板+图文游记，支持分类筛选 |
| trip-template-publish.js | 行程详情页"发布到我的专栏"流程 |
| travel-note.js | 图文游记的发布、查看、列表展示 |
| note-editor.js | Quill 编辑器封装，含自定义 blot（地点标注、行程挂载卡） |
| user-profile.js | 个人主页，聚合展示作者所有内容与数据 |

### 3.4 modules/creator-system/ — 创作者价值体系
| 文件 | 职责 |
|------|------|
| points-ledger.js | 积分账本：记录每笔积分变动，支持查询与对账 |
| level-system.js | 等级计算、升级触发、功能特权解锁与校验 |
| newbie-tasks.js | 新手任务清单状态管理、完成检测、经验发放 |
| voting.js | 每日签到得票、投票、票数统计、曝光加权计算 |
| items.js | 道具获取（活动/掉落）、收藏柜展示、活动任务进度 |

### 3.5 modules/checkin/ — 吐槽打卡系统
| 文件 | 职责 |
|------|------|
| checkin-action.js | 实地打卡（GPS 校验 500m）、云打卡（手动选地点）、打卡记录 |
| roast.js | 吐槽内容发布（图文）、查看、点赞收藏互动 |
| poi-topic.js | 地点专题区域聚合页：地图小窗+吐槽瀑布流+头像墙 |
| moderation.js | 举报接收、审核队列、违规分级处理、自动关键词过滤 |

### 3.6 modules/pdf-export/ — PDF 导出
| 文件 | 职责 |
|------|------|
| pdf-export.js | 行程详情/打卡内容 → HTML → PDF（jsPDF + html2canvas） |

## 4. 模块依赖关系

```
                    ┌─────────────┐
                    │  lib/       │
                    │  supabase   │
                    │  amap       │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  shared/    │

                    │  ui-kit     │
                    │  migrate    │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
┌───────▼───────┐ ┌────────▼────────┐ ┌───────▼───────┐
│  community/   │ │  creator-system/│ │   checkin/    │
│  专栏         │ │  价值体系       │ │   打卡吐槽    │
└───────┬───────┘ └────────┬────────┘ └───────┬───────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
                    ┌──────▼──────┐
                    │  pdf-export │
                    └─────────────┘
```

**依赖规则：**
- `lib/` 是底层，不依赖任何模块
- `shared/` 依赖 `lib/`，不依赖 `modules/`
- `modules/` 依赖 `lib/` 和 `shared/`，模块间尽量不直接依赖（通过事件总线通信）
- `pdf-export` 可被任意模块调用

## 6. 模块间通信

模块间**不直接调用**，通过全局事件总线（Event Bus）通信：

```js
// shared/ui-kit.js 内置 EventBus
window.EventBus = {
  on(event, cb) { ... },
  off(event, cb) { ... },
  emit(event, data) { ... }
};

// 示例：行程发布后通知社区刷新
EventBus.emit('template:published', { id, title });
EventBus.on('template:published', () => CommunityPage.refresh());
```

这样模块间解耦，单个模块可独立开发与测试。
