## 1. 代码风格总则

- 纯原生 HTML/CSS/JS，**不引入框架**（React/Vue 等）
- 使用 ES6+ 语法（const/let、箭头函数、模板字符串、解构、async/await）
- 每个模块用 **IIFE 包裹**，只暴露必要的全局对象，避免污染全局命名空间
- 缩进：2 空格，不混用 Tab
- 行尾不加分号（与现有代码风格一致）
- 文件编码统一 UTF-8，换行符 LF

## 3. 模块结构规范

每个功能模块文件统一采用 IIFE + 命名空间模式：

```js
// ✅ 正确：模块封装
(function (global) {
  const ModuleName = {
    state: { ... },
    async init() { ... },
    async _privateMethod() { ... }
  }
  global.ModuleName = ModuleName
})(window)

// ❌ 错误：裸露全局变量
let count = 0
function doSomething() { ... }
```

**模块必须暴露的标准接口：**
- `init()`：初始化（绑定事件、加载数据）
- `destroy()`：清理（解绑事件、释放资源），页面切换时调用
- 模块状态通过 `state` 对象集中管理，不散落全局变量

## 4. Supabase 使用规范

### 4.1 客户端获取
```js
// ✅ 正确：统一从 lib/supabase-client.js 获取
const { data, error } = await supabase.from('trip_templates').select('*')

// ❌ 错误：自己创建客户端实例
const client = createClient(url, key)  // 禁止
```

### 4.2 错误处理
```js
// ✅ 正确：所有 Supabase 调用必须处理 error
const { data, error } = await supabase.from('trip_templates').select('*')
if (error) {
  UiKit.toast('加载失败，请重试', 'error')
  console.error('[TripTemplate] 加载失败:', error.message)
  return
}
// 使用 data...

// ❌ 错误：忽略 error
const { data } = await supabase.from('trip_templates').select('*')
render(data)  // error 时 data 为 null，直接崩
```

### 4.3 RLS 策略
- 所有表必须启用 Row Level Security
- 公共内容（行程模板、图文游记、吐槽）：所有人可读，仅作者可写
- 私密数据（积分账本、关注关系、道具）：仅本人可读写
- 写操作必须通过 `auth.uid()` 校验当前用户

### 4.4 查询规范
```js
// ✅ 正确：明确指定字段，分页查询
const { data } = await supabase
  .from('trip_templates')
  .select('id, title, author_id, likes, copies, favorites')
  .range(0, 19)
  .order('created_at', { ascending: false })

// ❌ 错误：select('*') 拉全表
const { data } = await supabase.from('trip_templates').select('*')
```

## 5. 异步与错误处理

### 5.1 异步操作必须有 loading 态
```js
// ✅ 正确
UiKit.showLoading()
try {
  const data = await fetchTemplates()
  render(data)
} catch (e) {
  UiKit.toast('加载失败', 'error')
} finally {
  UiKit.hideLoading()
}
```

### 5.2 边界值校验
```js
// ✅ 正确：null 安全
const author = template?.author_id
if (!author) return

// ❌ 错误：假设数据存在
const author = template.author_id  // template 为 null 时崩溃
```

## 7. AI 协作协议（重要）

> 本项目由 AI 辅助开发，以下规则 AI 必须严格遵守。

1. **先读文档再动手**：每次开发前必须读取 `specs/` 下相关文档，了解产品概述、技术栈、目录结构
2. **不改现有代码逻辑**：修改 `js/`、`css/` 下现有文件时，只做增量添加（新按钮、新入口），不重构现有逻辑
3. **新功能放 modules/**：所有社区化升级的新功能代码必须放在 `modules/` 对应目录下
4. **遵循模块结构**：新模块必须用 IIFE 封装，暴露 `init()` 和 `destroy()`
5. **Supabase 调用必处理 error**：禁止忽略 error
6. **不引入新框架**：保持纯原生，第三方库只通过 CDN 引入且需经确认
7. **中文注释**：所有注释用中文，说明"为什么"而非"是什么"
8. **不过度设计**：只实现当前需求，不加未来可能用到的参数和抽象
9. **数据安全**：写操作必须校验当前用户身份，不信任前端传入的 user_id
10. **保留硬约束**：遵循 project_memory 中的所有硬约束（如 AI 不限次、10分钟时间粒度、POI 后端过滤等）

## 9. 注释规范

```js
/**
 * 计算创作者积分（含防刷校验）
 * @param {string} userId - 创作者 ID
 * @param {'view'|'like'|'favorite'|'adopt'|'follow'} action - 行为类型
 * @param {string} targetId - 目标内容 ID
 * @returns {Promise<number>} 本次实际增加的积分（防刷命中返回 0）
 */
async function calcPoints(userId, action, targetId) { ... }
```

- 函数注释说明"为什么这样做"，不是"做了什么"
- 复杂逻辑块用行内注释 `// 说明`
- 明显的代码不加注释
