## 1. 功能概述

为行程列表增加**标签分类**功能（类似 OPPO 便签的集合机制），用户可自建标签对行程归类管理。顶部横向标签栏可快速切换查看不同分类下的行程，解决行程过多时难以管理和查找的问题。

### 1.1 核心概念

- **标签**：用户自建的分类集合，如"休闲出行""商务出差""亲子游"等
- **一个行程只能归属一个标签**（或无标签，归入"全部"）
- **全部**：系统默认项，存放所有未打标签的行程；同时也是查看全部行程的入口
- **最近删除**：系统默认项，存放被删除标签连带删除的行程，7天后永久清除

### 1.2 与现有功能的关系

- 标签分类**只管"我的行程"tab**，不影响"旅行记忆"tab（旅行记忆继续用于挑行程导出PDF）
- 现有的行程卡片折叠、回到顶部按钮、长按选择模式保持不变
- 现有的排序逻辑（进行中→待出发→已完成→已归档）在标签分类内继续生效

## 2. 用户故事

### US-1：查看标签分类
作为一个行程较多的用户，我希望顶部有一排横向标签栏，点某个标签就只看这个分类下的行程，这样我不用在长长的列表里翻找。

### US-2：给行程打标签
作为一个用户，我希望在行程卡片上和行程详情页里都能给行程设置标签，方便我把行程归到对应的分类。

### US-3：管理标签
作为一个用户，我希望有一个专门的标签管理页面，可以新建、重命名、删除标签，还能拖动调整标签的排列顺序。

### US-4：安全删除标签
作为一个用户，当我删除一个标签时，系统会弹窗提醒我标签下的行程怎么处理，避免误删重要行程。

### US-5：找回误删行程
作为一个用户，我希望被删除的行程能暂存在"最近删除"里保留7天，期间可以恢复，避免误操作导致永久丢失。

## 3. 功能详细需求

### 3.1 标签栏（TripTagBar）

**位置**：行程列表页"我的行程"tab 内，位于 tab 栏下方、行程列表上方。

**布局**：
- 横向滚动条，类似 OPPO 便签
- 从左到右固定顺序：`全部` → 用户标签（按创建时间排序）→ `最近删除`
- "全部"始终在最左侧并默认选中
- "最近删除"始终在最右侧，用特殊样式区分（如灰色/删除图标）

**交互**：
- 点击标签 → 切换显示该标签下的行程，当前选中标签高亮
- 点击"全部" → 显示所有行程（含已打标签的）
- 横向滑动浏览更多标签

### 3.2 给行程设置标签

**入口一：行程卡片上**
- 在行程卡片操作区或长按菜单中，提供"设置标签"入口
- 点击后弹出标签选择面板（底部弹窗），列出所有标签 + "无标签（移到全部）"选项
- 选择后立即生效，卡片可显示当前标签名

**入口二：行程详情页内**
- 行程详情页增加标签选择器
- 操作方式同上

**规则**：
- 一个行程只能有一个标签
- 新建行程默认无标签，归入"全部"
- 换标签 = 直接选新标签覆盖旧的，无需先取消

### 3.3 标签管理页面

**入口**：标签栏右侧或行程页右上角设置入口，进入标签管理页。

**功能**：
- **新建标签**：输入标签名，可选输入 emoji/颜色（选做，MVP 可只输入名称）
- **重命名**：点击标签可编辑名称
- **删除**：见 3.4 删除规则
- **排序**：长按拖动调整标签顺序（排序结果同步反映到顶部标签栏）

### 3.4 删除标签规则

删除标签时弹出确认弹窗，提供三个选项：

| 选项 | 说明 |
|------|------|
| **删除标签并保留行程** | 标签删除，标签下的行程退回"全部"（变为无标签） |
| **删除标签及全部行程** | 标签和行程一起删除，行程进入"最近删除"，7天后永久删除 |
| **取消** | 不执行任何操作 |

> 说明：用户原话提到"删除标签就对应删掉合集中所有内容→放最近删除→7天永久删除/直接全部删除/全部恢复"。经澄清，增加"删除标签但保留行程"选项，避免用户只想整理标签时误删行程。

### 3.5 最近删除

**位置**：标签栏最右侧固定项，点击进入"最近删除"视图。

**展示**：
- 列表展示所有被删除的行程
- 每条行程显示：行程标题、目的地、删除日期、**剩余天数**（"5天后永久删除"）
- 剩余天数 = 7 - (当前日期 - 删除日期)，≤0 时自动永久删除

**操作**：
- **全部恢复**：一键恢复所有行程到"全部"（无标签状态）
- **全部删除**：一键永久删除所有行程（二次确认）
- **单条恢复**：点击单条行程可恢复到"全部"
- **单条永久删除**：左滑/长按单条可永久删除

**自动清理**：
- 应用启动时检查"最近删除"中的行程，超过7天的自动永久删除

### 3.6 行程卡片标签显示

- 已打标签的行程卡片上显示当前标签名（如小标签/chip 形式）
- 未打标签的行程不显示标签标识
- 在"全部"视图下，标签标识帮助用户快速识别行程归属

## 4. 数据模型（概要）

> 详细字段设计在技术方案阶段（Skill 9）细化。

### 4.1 标签数据

```
trip_tags: [
  { id, name, emoji?, createdAt, order }
]
```
- 存储在 localStorage，key: `trip_tags`
- `order` 字段用于排序，与 `createdAt` 初始一致，用户拖动后更新

### 4.2 行程-标签关联

在现有 trip 对象上新增字段：
```
trip.tagId: string | null  // 标签ID，null表示无标签（在"全部"中）
```

### 4.3 最近删除数据

```
trip_recently_deleted: [
  { ...tripData, tagId, deletedAt: timestamp }
]
```
- 存储在 localStorage，key: `trip_recently_deleted`
- `deletedAt` 用于计算7天倒计时

## 5. 验收标准（AC）

### AC-标签-1：查看标签分类下的行程
- **Given**：用户已创建"休闲出行"标签，且有3个行程归在该标签下
- **When**：点击标签栏中的"休闲出行"
- **Then**：列表只显示这3个行程，"休闲出行"标签高亮选中
- **Verification**：human-judgment

### AC-标签-2：从"全部"查看所有行程
- **Given**：用户有多个标签，各标签下有行程
- **When**：点击标签栏中的"全部"
- **Then**：列表显示所有行程（含已打标签的），每个卡片显示其标签标识
- **Verification**：human-judgment

### AC-标签-3：给行程设置标签
- **Given**：用户在"全部"中有一个未打标签的行程
- **When**：在行程卡片上点击"设置标签"，选择"休闲出行"
- **Then**：行程卡片显示"休闲出行"标签标识，该行程出现在"休闲出行"标签视图中
- **Verification**：human-judgment

### AC-标签-4：给行程换标签
- **Given**：一个行程已有"休闲出行"标签
- **When**：在行程详情页将标签改为"商务出差"
- **Then**：行程从"休闲出行"标签视图消失，出现在"商务出差"标签视图中
- **Verification**：human-judgment

### AC-标签-5：新建标签
- **Given**：用户在标签管理页面
- **When**：点击"新建标签"，输入"亲子游"，确认
- **Then**：标签栏出现"亲子游"标签，按创建时间排在已有标签之后
- **Verification**：human-judgment

### AC-标签-6：重命名标签
- **Given**：用户在标签管理页面，有一个"休闲出行"标签
- **When**：点击重命名，改为"周末休闲"，确认
- **Then**：标签栏和所有相关行程卡片上的标签名同步更新为"周末休闲"
- **Verification**：human-judgment

### AC-标签-7：标签拖动排序
- **Given**：用户在标签管理页面，有多个标签
- **When**：长按"商务出差"标签拖动到"休闲出行"前面，松手
- **Then**：标签栏中"商务出差"排在"休闲出行"前面，顺序持久化
- **Verification**：human-judgment

### AC-标签-8：删除标签但保留行程
- **Given**：用户要删除"休闲出行"标签，该标签下有3个行程
- **When**：点击删除，弹窗选择"删除标签并保留行程"
- **Then**："休闲出行"标签从标签栏消失，3个行程退回"全部"（无标签状态），卡片上不再显示标签标识
- **Verification**：human-judgment

### AC-标签-9：删除标签及行程进入最近删除
- **Given**：用户要删除"商务出差"标签，该标签下有2个行程
- **When**：点击删除，弹窗选择"删除标签及全部行程"
- **Then**："商务出差"标签消失，2个行程从列表消失，进入"最近删除"，每条显示"7天后永久删除"
- **Verification**：human-judgment

### AC-标签-10：最近删除-单条恢复
- **Given**："最近删除"中有1个行程，显示"5天后永久删除"
- **When**：点击该行程的恢复按钮
- **Then**：行程从"最近删除"消失，回到"全部"列表中（无标签状态）
- **Verification**：human-judgment

### AC-标签-11：最近删除-全部恢复
- **Given**："最近删除"中有3个行程
- **When**：点击"全部恢复"
- **Then**：3个行程全部回到"全部"列表，"最近删除"清空
- **Verification**：human-judgment

### AC-标签-12：最近删除-7天自动清理
- **Given**："最近删除"中有1个行程，删除日期为8天前
- **When**：应用启动
- **Then**：该行程被自动永久删除，不再出现在"最近删除"中
- **Verification**：programmatic

### AC-标签-13：剩余天数正确显示
- **Given**："最近删除"中有一个行程，删除于3天前
- **When**：进入"最近删除"视图
- **Then**：该行程显示"4天后永久删除"（7-3=4）
- **Verification**：programmatic

## 6. 非目标（Out of Scope）

- 一个行程归属多个标签（明确只支持单标签）
- 标签颜色/emoji 自定义（MVP 只做名称，颜色后续可扩展）
- 标签的智能推荐/自动分类
- "旅行记忆"tab 的标签分类（标签只管"我的行程"）
- 云端同步标签数据（MVP 先用 localStorage，后续随 Supabase 迁移）

---

## 7. 约束与假设

### 约束
- 前端主体保留 HTML/CSS/JS，不引入框架
- 标签数据存储在 localStorage
- 行程数据结构沿用现有 `trip_history`（对象 map），在 trip 对象上新增 `tagId` 字段

### 假设
- 用户标签数量不会太多（通常 5-10 个），横向滚动足够
- "最近删除"中的行程数据量小，不需要分页
